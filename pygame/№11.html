Проект «Нарисуй рисунок»

Итоговый проект по теме "Циклы в Python": создайте консольную программу, которая рисует ASCII‑арт

Постановка задачи

Вам предстоит разработать программу, которая с помощью циклов for и/или while рисует ASCII‑рисунок в консоли.

Что нужно сделать

Выберите один из предложенных вариантов (или придумайте свой) и напишите код, который строит рисунок из символов. Рисунок должен содержать повторяющиеся элементы или симметрию, для формирования которых обязательно используются циклы (в том числе вложенные). Программа должна запрашивать у пользователя размер (например, высоту) и выводить рисунок.

Основные требования

  • Использовать вложенные циклы (for внутри for или while внутри for/while)
  • Применять range() для перебора строк и столбцов
  • Разрешается использовать break / continue для изменения логики
  • Код должен быть понятным, с комментариями к ключевым участкам
  • Рисунок должен меняться в зависимости от введённого размера (не фиксированный)

Пример реализации треугольника

Ниже приведён пример кода для рисования треугольника. Используйте его как основу или отправную точку.

h = int(input("Введите высоту треугольника: "))
for i in range(1, h + 1):
    # Печатаем i звёздочек
    for j in range(i):
        print("*", end="")
    print() # Переход на новую строку

Требования к реализации

Технические требования

  • Программа написана на Python 3
  • Использован хотя бы один вложенный цикл
  • Размер рисунка задаётся переменной или вводом пользователя
  • Код не содержит «скопированных» строк без циклов (не должно быть повторяющихся print для увеличения размера)
  • Допускается использование функций, но не обязательно

Критерии качества

  • Рисунок симметричен или логично повторяется
  • Код структурирован и читаем
  • Есть комментарии к ключевым циклам
  • Программа не завершается ошибкой при любых корректных входных данных
  • Названия переменных осмысленны

Чеклист перед сдачей

Прежде чем отправить проект, убедитесь, что:

Работоспособность

  • Программа запускается и запрашивает размер
  • Рисунок корректно отображается для разных размеров (например, 3, 5, 10)
  • При вводе отрицательного или нулевого размера выводится сообщение об ошибке или программа завершается без аварий

Код

  • Нет синтаксических ошибок
  • Вложенные циклы используются правильно
  • Циклы не бесконечные (если не предусмотрено логикой)
  • Присутствуют комментарии

Оформление

  • Код отформатирован (отступы одинаковы)
  • Название файла соответствует заданию (например, drawing.py)
  • В начале файла указан автор и краткое описание рисунка
