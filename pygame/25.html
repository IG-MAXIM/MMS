Проект «Крестики-нолики»

Создайте графическую игру на Tkinter с сеткой из кнопок и проверкой победителя

Постановка задачи

Разработайте игру «Крестики-нолики» для двух игроков с графическим интерфейсом. Игроки по очереди нажимают на кнопки поля 3×3, после каждого хода программа проверяет, не завершилась ли игра победой одного из игроков или ничьей.

Основные функции

  • Окно с заголовком «Крестики-нолики»
  • Сетка 3×3 из девяти кнопок, созданных с помощью цикла и размещённых через grid
  • При нажатии на свободную клетку она заполняется символом текущего игрока («X» или «O»)
  • После каждого хода проверяется наличие выигрышной комбинации (три в ряд по горизонтали, вертикали или диагонали) или заполненность всех клеток
  • При завершении игры (победа или ничья) на экран выводится сообщение, а поле блокируется
  • Кнопка «Новая игра» для сброса поля и начала заново

Требования к реализации

Технические требования

  • Использовать библиотеку tkinter
  • Кнопки поля создавать в цикле (не вручную каждую)
  • Хранить состояние игры в списке списков (матрице 3×3) — глобальная переменная
  • Хранить текущего игрока в глобальной переменной (строка или флаг)
  • Каждая кнопка через command вызывает одну общую функцию, которая определяет номер клетки (можно через lambda с передачей индексов)
  • Реализовать отдельную функцию проверки победы, которая возвращает True/False и символ победителя
  • Реализовать функцию проверки ничьей (заполнена ли матрица)
  • Кнопка «Новая игра» сбрасывает матрицу, очищает надписи на кнопках и активирует их
  • При победе или ничьей выводить сообщение с помощью messagebox.showinfo

Критерии качества

  • Плавная и понятная игровая логика
  • Кнопки не реагируют на повторные нажатия в занятых клетках
  • После завершения игры кнопки блокируются (можно через config(state=DISABLED))
  • Программа не завершается аварийно при любых действиях пользователя
  • Код разбит на логические функции, есть комментарии к основным блокам

Чеклист перед сдачей

Перед отправкой проекта убедитесь, что:

Игровой процесс

  • Игра запускается и отображается окно с сеткой 3×3
  • Игроки ходят по очереди, символы корректно отображаются на кнопках
  • Нельзя занять уже занятую клетку
  • Программа правильно определяет победу (горизонталь, вертикаль, диагонали)
  • Определяется ничья, когда все клетки заполнены без победителя
  • После победы/ничьи дальнейшие ходы невозможны, выводится сообщение

Код

  • Кнопки созданы в цикле, не вручную
  • Функции check_win и is_draw работают корректно
  • Использованы глобальные переменные и lambda для передачи индексов
  • Кнопка «Новая игра» сбрасывает всё состояние и активирует клетки
  • Код отформатирован, содержит комментарии
