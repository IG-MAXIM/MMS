Спрайты, перемещение и простейшие столкновения

Создайте собственные игровые объекты с помощью классов и обработайте столкновения между ними

Что такое спрайт?

Спрайт — это графический объект в игре (персонаж, враг, препятствие). В Pygame спрайты удобно создавать с помощью классов, наследуя pygame.sprite.Sprite.

class Player(pygame.sprite.Sprite):
    def __init__(self, x, y):
        super().__init__()
        self.image = pygame.Surface((50, 50))
        self.image.fill((0, 255, 0))
        self.rect = self.image.get_rect(topleft=(x, y))

Каждый спрайт имеет image (поверхность с картинкой) и rect (прямоугольник для позиционирования и столкновений).

Перемещение объекта

Для движения спрайт обновляет координаты своего rect. Можно менять rect.x, rect.y или использовать готовый метод rect.move(dx, dy).

def update(self):
    keys = pygame.key.get_pressed()
    if keys[pygame.K_LEFT]:
        self.rect.x -= self.speed
    if keys[pygame.K_RIGHT]:
        self.rect.x += self.speed
    # и так далее...

Обычно метод update() вызывается каждый кадр для каждого спрайта.

Совет: используйте pygame.time.Clock() для ограничения FPS, чтобы движение было одинаковым на разных компьютерах.

Группы спрайтов и столкновения

Pygame предоставляет класс pygame.sprite.Group для управления множеством спрайтов. С помощью групп легко проверять столкновения.

Группа всех спрайтов

all_sprites = pygame.sprite.Group()
player = Player(100, 100)
all_sprites.add(player)
# В цикле:
all_sprites.update()
all_sprites.draw(screen)

Проверка столкновений

# Столкновение спрайта с группой
hit = pygame.sprite.spritecollide(player, enemy_group, False)
if hit:
    # Обработка столкновения

spritecollide(sprite, group, dokill) возвращает список спрайтов из группы, с которыми пересекся данный спрайт. Если dokill=True, столкнувшиеся спрайты удаляются из группы.

Практические задания

Задание 1: Класс Квадрат

Создайте класс Square (наследник Sprite), который рисует синий квадрат 30×30 в позиции (x, y) и имеет метод update() для горизонтального движения (по стрелкам влево/вправо).

class Square(pygame.sprite.Sprite):
    def __init__(self, x, y):
        super().__init__()
        self.image = pygame.Surface((30, 30))
        self.image.fill((0, 0, 255))
        self.rect = self.image.get_rect(topleft=(x, y))
        self.speed = 5
    def update(self):
        keys = pygame.key.get_pressed()
        if keys[pygame.K_LEFT]:
            self.rect.x -= self.speed
        if keys[pygame.K_RIGHT]:
            self.rect.x += self.speed

Задание 2: Сбор предметов

Создайте игрока (зелёный квадрат) и несколько врагов (красные круги), разбросанных по экрану. При столкновении игрока с врагом враг должен исчезать, а счёт увеличиваться. Используйте pygame.sprite.spritecollide.

# Фрагмент игрового цикла
all_sprites.update()
# Проверка столкновений игрока с врагами
hits = pygame.sprite.spritecollide(player, enemies, True)
for hit in hits:
    score += 10
    new_enemy = Enemy() # создаём нового врага
    all_sprites.add(new_enemy)
    enemies.add(new_enemy)
all_sprites.draw(screen)

Задание 3: Избегание препятствий

Игрок (синий квадрат) движется по стрелкам. На экране есть неподвижные красные препятствия (Sprites). Если игрок касается препятствия, игра останавливается, и выводится сообщение о проигрыше. Используйте spritecollide с dokill=False.

# В игровом цикле после обновления
hits = pygame.sprite.spritecollide(player, obstacles, False)
if hits:
    game_over = True
    font = pygame.font.Font(None, 74)
    text = font.render("Game Over", True, (255, 0, 0))
    screen.blit(text, (200, 250))
    pygame.display.flip()
    pygame.time.wait(2000)
    running = False

Итог урока

Теперь вы умеете:

  • Создавать спрайты как классы, наследующие pygame.sprite.Sprite
  • Управлять перемещением через обновление rect в методе update()
  • Объединять спрайты в группы для коллективного обновления и отрисовки
  • Обнаруживать столкновения с помощью pygame.sprite.spritecollide
Ключевой навык: Объектно-ориентированный подход к игровым сущностям и обработка коллизий — фундамент любой настоящей игры. Теперь вы можете создавать игры с собиранием предметов, уклонением от врагов и простой физикой.
